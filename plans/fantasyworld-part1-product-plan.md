# FantasyWorld Part 1 功能方案

## Summary

FantasyWorld 第一版定位为“AI 叙事玩家”的单人世界推演游戏，不是普通聊天应用。核心体验是：用户作为导演/GM 创建一个可编辑的世界存档，点击回合推进，让世界、角色、关系和事件自动演化，并在必要时用自然语言介入、修正、回滚。

15 分钟内的成功时刻：玩家创建世界后推进几回合，看到角色因目标、秘密、关系和地点限制产生合理事件，并能作为 GM 介入调整世界。

## Key Product Decisions

- 核心循环：`创建存档 -> 推进回合 -> 查看事件和状态变更 -> 接受/修正/继续 -> 可回滚`。
- 用户身份：默认是导演/GM，可查看和编辑全局状态，不默认扮演单一角色。
- 推演节奏：回合制推进，不做实时自动模拟。
- 世界时间：每个存档记录回合序号，并允许设置抽象时间节奏，例如“每回合约一天/一幕/数小时”。
- 角色规模：每个存档首版支持约 3-8 个活跃推演角色。
- 推演方式：世界裁判统筹本回合焦点，再按需调用相关角色生成意图、行动、对话，最后统一结算事件。
- 角色自主性：按需思考行动，不要求所有角色每回合发言。
- 对话生成：事件触发，通常发生在同地点角色之间，避免退化成群聊。
- 游戏机制：轻规则叙事，使用少量状态、关系强度、目标进度和轻量随机，由 LLM 裁定主要结果。
- 随机性：每个存档有“稳定到意外”的随机性设置，默认中低随机；随机结果需要可回滚、可调试。
- 内容边界：每个存档可自定义内容限制，默认 PG-13 冒险冲突风格；首版按软风格偏好处理，不作为硬失败拦截规则。

## Core Features And Interfaces

- 页面范围：
  - 存档列表页：新建、继续、导入、导出存档。
  - 创建向导：选择 3-5 个通用世界模板，填写自由设定、世界规则、角色种子、内容边界和存档语言；提交后生成初始世界草稿，用户预览、编辑、确认后才成为正式存档。
  - 主推演界面：桌面优先三栏工作台，左侧世界/地点/角色导航，中间事件时间线，右侧选中角色、事件或状态详情。
  - 模型配置页：用户自配 API key、host、model；提供全局默认配置，存档可覆盖。
- 存档语言：UI 支持中英双语；每个存档选择一种生成语言，同一存档默认保持语言一致。
- 世界状态首版结构化对象：
  - `Character`：姓名、简介、性格、长期目标、短期目标、当前位置、状态、秘密、私有记忆。
  - `Location`：名称、描述、当前状态、在场角色。
  - `Relationship`：角色双方、关系标签、强度、备注摘要。
  - `WorldMemory`：全局时间线、当前世界摘要、地点摘要。
  - `Turn`：回合事件、角色行动、对话片段、状态变更、模型调用摘要。
  - `ModelConfig`：全局默认配置和存档覆盖配置；首版默认单模型，预留裁判/角色/摘要模型分工。
- GM 世界编辑：
  - GM 可随时编辑角色、地点、关系、世界记忆、角色私有记忆、秘密、目标、内容边界、随机性和存档模型覆盖。
  - 回合草稿编辑只用于修正当前回合结果，不承担全部世界编辑职责。
- 记忆系统：
  - 使用分层摘要加角色私有记忆。
  - GM 默认可查看和编辑全部记忆、秘密、目标、关系和状态。
  - 回合结束自动写入记忆和摘要，但用户可在接受前编辑，也可通过回滚撤销。
- 回合结果呈现：
  - 中间主视图显示事件日志。
  - 旁边显示角色状态、目标变化、关系变化、记忆变化和调用摘要。
  - 每回合必须明确列出状态变更，而不是只藏在叙事文本里。
- 用户介入：
  - 主要通过自然语言 GM 指令，例如“让暴风雪袭击港口”。
  - 回合结束后提供接受、修正、继续。
  - 修正范围包括事件文本、状态变更、角色记忆、目标和关系。
- 历史能力：
  - 保存回合历史和快照。
  - 首版支持回滚，不做完整分支树。
  - 支持 JSON 导入/导出完整可玩存档。

## Test And Acceptance Scenarios

- 创建存档：用户选择模板、补充世界设定、添加角色种子后，系统生成初始世界草稿；用户确认后，正式存档包含 3-8 个角色、地点、关系和初始世界摘要。
- 推进回合：点击推进后，系统只选择相关角色行动，生成事件、必要对话、状态变更和记忆更新。
- 介入世界：用户输入 GM 指令后，下一回合事件和角色行为应受该指令影响，同时不违反世界规则。
- 编辑世界：GM 可在非回合生成期间编辑角色、地点、关系、记忆、秘密、目标和存档设置，后续回合使用编辑后的状态。
- 记忆一致性：角色后续行动应参考自己的目标、私有记忆、秘密、关系和地点限制。
- 修正闭环：用户能编辑一次回合结果中的事件和状态变更，接受后后续回合使用修正后的状态。
- 回滚闭环：用户能回到前一回合，世界状态、记忆、关系和时间线恢复到对应快照。
- 双语体验：UI 可中英切换；中文存档生成中文内容，英文存档生成英文内容。
- 成本透明：每回合显示模型、调用次数、耗时和 token 估算或实际统计。

## Assumptions And Defaults

- 第一版优先做“可玩闭环”，不追求完整百科系统、多人协作、实时自动模拟或复杂数值规则。
- 技术栈、数据库、认证、部署、API key 存储安全策略以 `fantasyworld-tech-stack-plan.md` 为准。
- 世界模板首版做少量通用模板，不只限传统奇幻题材。
- 风格控制通过“风格指令区”提供，不开放完整系统提示词编辑。
- 主体验桌面优先，移动端先保证可用适配。
